## 重构
为了实现变更，不可避免的要违反最初的设计构架。经过一段时间以后，软件的架构就千疮百孔了。bug越来越多，越来越难维护，新的需求越来越难实现，软件的架构对新的需求渐渐的失去支持能力。

通过重构，不断的调整系统的结构，使系统对于需求的变更始终具有较强的适应能力。让项目主框架突出鲜明，给人一种思路清晰，一目了然的感觉，其实重构是对框架的一种维护。

* 重构时机
* 间接层的利弊
* 重构与设计是互补的，设计应该是适度的设计，而不必过度的设计.
