## 迭代改进
“贪婪技术的一种竞争方案”，都只能应用于最优问题。

##### “迭代改进” 与 “贪婪技术” 的差异？
```
* 相同点： 都是 step-by-step 的执行。
* 贪婪技术： 总是把一个局部最优解加入到一个部分构造解之中。 F = F(s1) + F(s2) + ...。
* 迭代改进： 每一步都是对现有解的改进。 F = F + delta(s)。
```
请注意： 贪婪技术专门强调了，其存在的“不可取消”特性。 选择一旦做出，在后面的步骤中就无法改变了。
